# quartic-mean-field-hamiltonian

Code repository accompanying the publication entitled "On the dynamical and statistical properties of a quartic mean field Hamiltonian model".

This project contains the code to generateaall the data from all the figures. Please, keep in mind that most simulations in this project take days to months to finish.

## Requirements

### Compiler

A C compiler supporting C99 or newer, tested with:

- GCC
- Intel oneAPI icx
- Apple Clang (macOS)

The compiler can be selected via

```bash
make CC=gcc
make CC=cc
make CC=icx
```

### Operating system

- Linux
- macOS

The build system automatically selects appropriate system libraries based on the OS.

Windows is not supported natively. Users on Windows can run the code by installing a Linux environment using one of the following options:

- installing a Linux distribution (e.g. [Ubuntu](https://ubuntu.com)) as a dual-boot system,
- using Windows Subsystem for Linux (WSL2), or
- running a Linux virtual machine (e.g., VirtualBox or VMware).

### External libraries

This project depends on the GNU Scientific Library (GSL), including

- libgsl
- libglscblas

GSL must be installed system-wide and discoverable by the compiler.

- macOS (Homebrew)

```bash
brew install gsl
```

- Linux (Debian/Ubuntu)

```bash
sudo apt install libgsl-dev
```

### Linear algebra and math libraries

The following libraries are linked automatically depending on the platform:

- macOS:
  - Accelerate framework (-framework Accelerate)
  - Standard math library (libm)

- Linux:
  - LAPACK (liblapack)
  - Standard math library (libm)

### Build system

- GNU Make

## Build and Usage

### Building the project

From the root directory of the project, simply run:

```bash
make
```

This will compile all executables defined in `programs/` and place the resulting binaries in the `bin/` directory, which is created automatically if it does not already exist.

To build a specific executable only, run:

```bash
make <program_name>
```

For example:

```bash
make lle_history
make q_history
```

The corresponding executables will be generated in `bin/`.

### Compiler selection

By default, the project uses `gcc`. A different compiler can be selected at build time, for example:

```bash
make CC=icx
make q_history CC=icx
```

The recommended compiler is `icx`, as it consistently produces better optimized binaries and shorter execution times for this project. Other compilers such as `gcc` and `clang` may result in slightly lower performance.

### Running the executables

All executables are located in the `bin/` directory and can be run from the project root, for example:

```bash
./bin/q_history.x
```

All programs accept command-line arguments specifying system size, energy, integration time, and other simulation parameters. The expected arguments and usage can be obtained by running the executable without arguments or by consulting the corresponding source file in `programs/`.

For instance, the above command results in:

```text
Usage: ./bin/q_history.x <dof> <eps> <total_time> <seed>

Arguments:
  <dof>         Number of degrees of freedom (positive integer)
  <eps>         Specific energy Оµ = E / N
  <total_time>  Total integration time
  <seed>        Random number generator seed
```

The correct way of executing it should then be

```bash
./bin/q_history.x 1024 100 100000000 1313
```

### Cleaning the build

To remove all generated executables, run:

```bash
make clean
```

## Reproducing the results from the paper

The numerical data presented in the paper are generated using the shell scripts provided in the `scripts/` directory. Each script corresponds to a specific figure in the manuscript.

To generate the data, run the desired script from the project root directory:

```bash
chmod +x scripts/*.sh
./scripts/fig01.sh
./scripts/fig02.sh
./scripts/fig03.sh
./scripts/fig04.sh
./scripts/fig05.sh
```

The scripts compile the required executables and execute the corresponding simulations, writing the output data to the `data/` directory.

> **_NOTE_**: Most simulations in this project are computationally expensive. Depending on the system size, individual runs may take from several days to several months to complete on standard hardware.

The recommendation is:

- run the scripts on high-performance computing (HPC) systems when possible,
- carefully inspect and adjust the simulation parameters before execution,
- test the scripts with reduced system sizes or shorter integration times to validate the workflow.
